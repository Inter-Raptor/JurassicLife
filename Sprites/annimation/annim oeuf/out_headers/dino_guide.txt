=== Sprite Animations - Guide d'utilisation ===

Header à inclure : dino_anims_50x50.h
Personnage       : dino
Namespace C++     : dino

Taille d'une frame: 50 x 50 pixels
Couleur KEY (RGB565) pour transparence : 0xF81F
Trim mode         : none

Constantes disponibles :
 - dino::W   (largeur)
 - dino::H   (hauteur)
 - dino::KEY (couleur transparente)

Animations disponibles :
 - oeuf : 4 frame(s)
    enum id      : dino::ANIM_OEUF
    frames table : dino::dino_oeuf_frames  (pointeurs vers frames)
    frame count  : dino::dino_oeuf_count

Lecture générique (table globale) :
 - dino::ANIMS[animId].frames
 - dino::ANIMS[animId].count

Exemple minimal (LovyanGFX) :

  #include "dino_anims_50x50.h"
  // ... LGFX lcd;
  
  void drawAnimFrame(int x, int y, dino::AnimId anim, uint8_t frameIndex) {
    // Récupère la description de l'anim
    dino::AnimDesc ad;
    memcpy_P(&ad, &dino::ANIMS[anim], sizeof(ad));

    if (ad.count == 0) return;
    frameIndex %= ad.count;

    // Récupère le pointeur de frame
    const uint16_t* framePtr;
    memcpy_P(&framePtr, &ad.frames[frameIndex], sizeof(framePtr));

    // Dessin (color-key)
    // Note: pushImage n'a pas toujours le key, selon ta conf.
    // Si tu utilises un Sprite LovyanGFX, tu peux dessiner pixel par pixel en sautant KEY.
    lcd.pushImage(x, y, dino::W, dino::H, framePtr);
  }

Notes :
 - Les frames sont stockées en PROGMEM (flash).
 - Si tu veux une transparence parfaite (KEY), dessine en ignorant les pixels == KEY.
 - Pour animer: incrémente frameIndex toutes les X ms (ex: 80-120ms).
